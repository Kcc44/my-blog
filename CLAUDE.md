## Project

Tailwind Next.js Starter Blog — Next.js 15 + MDX + Tailwind CSS blog deployed to GitHub Pages at `https://kcc44.github.io/my-blog/` (repo `Kcc44/my-blog`). Fully localized to Chinese (zh-CN).

## Commands

```bash
yarn dev                 # Dev server (no basePath)
yarn build               # Production build (Contentlayer → Next.js)
yarn lint                # ESLint + Prettier fix
yarn analyze             # Build with bundle analyzer
```

### GitHub Pages local verification

```bash
EXPORT=1 UNOPTIMIZED=1 BASE_PATH=/my-blog yarn build
npx serve out
```

`BASE_PATH=/my-blog` on Windows Git Bash must be `BASE_PATH=//my-blog` to avoid MSYS2 path conversion. Not needed on GitHub Actions (Ubuntu).

## Architecture

**Content layer**: Contentlayer 2 (`contentlayer.config.ts`) generates typed JSON from `data/blog/**/*.mdx` into `.contentlayer/`. `tag-data.json` is auto-generated by the `onSuccess` hook.

**Dynamic routes with static export** (`output: 'export'`):
- `app/blog/[...slug]/page.tsx` — blog posts
- `app/blog/page/[page]/page.tsx` — paginated blog list
- `app/tags/[tag]/page.tsx` — posts by tag
- `app/tags/[tag]/page/[page]/page.tsx` — paginated tag listing

Each dynamic route must export `generateStaticParams`. `next.config.js` has `trailingSlash: true` for static HTML compatibility.

**Tag routing critical rule**: Do NOT use `encodeURI()` in `generateStaticParams` — use raw UTF-8 tag names as directory names. GitHub Pages decodes percent-encoded URL paths before filesystem lookup, causing 404 if the directory is named with percent-encoding.

**Key files**:
- `data/siteMetadata.js` — site config (URL, SEO, comments, analytics)
- `contentlayer.config.ts` — MDX types, remark/rehype plugins
- `scripts/rss.mjs` — RSS feed generation (post-build)
- `next.config.js` — basePath, trailingSlash, static export, CSP headers (comment only — `headers()` incompatible with export mode)
- `app/seo.tsx` — `genPageMetadata()` helper for per-page SEO
- `layouts/ListLayoutWithTags.tsx` — tag-filtered post list (shared by tag and blog pagination)
- `.github/workflows/pages.yml` — auto-deploy on push to `main`

**Writing new blog posts**: Create `data/blog/<slug>.mdx` with frontmatter (`title`, `date`, `tags`, `summary`). Push to `main` — GitHub Actions deploys automatically.
